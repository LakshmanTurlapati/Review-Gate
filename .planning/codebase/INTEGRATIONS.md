# External Integrations

**Analysis Date:** 2026-04-02

## APIs & External Services

**IDE & Local Protocols:**
- Cursor IDE / VS Code extension host - primary runtime for commands, webviews, notifications, file dialogs, and hotkeys implemented in `V2/cursor-extension/extension.js`
  - SDK/Client: built-in `vscode` module consumed by `V2/cursor-extension/extension.js`
  - Auth: None; runs inside the local Cursor installation declared by `V2/cursor-extension/package.json`
- Model Context Protocol over stdio - local tool surface exposed by `V2/review_gate_v2_mcp.py`
  - SDK/Client: `mcp>=1.9.2` from `V2/requirements_simple.txt`
  - Auth: None; process wiring is local through `V2/mcp.json` and installer-generated `~/.cursor/mcp.json` or `%USERPROFILE%\.cursor\mcp.json`

**Speech & Media:**
- SoX - microphone recording backend started from `V2/cursor-extension/extension.js`
  - SDK/Client: external `sox` CLI spawned through Node `child_process` in `V2/cursor-extension/extension.js`
  - Auth: OS-level microphone permission only
- Faster-Whisper - local speech transcription executed by `V2/review_gate_v2_mcp.py`
  - SDK/Client: `faster-whisper>=1.0.0` from `V2/requirements_simple.txt`
  - Auth: None; model execution is local

**UI Asset Delivery:**
- Font Awesome CDN - icon stylesheet embedded directly in the webview HTML generated by `V2/cursor-extension/extension.js`
  - SDK/Client: stylesheet URL to `cdnjs.cloudflare.com`
  - Auth: None

**Install & Distribution:**
- GitHub repository distribution - clone/install flow documented in `readme.md` and `V2/INSTALLATION.md`
  - SDK/Client: `git`
  - Auth: None for public clone
- Python package indexes - Python dependencies are installed by `pip` in `V2/install.sh`, `V2/install.ps1`, and `V2/install.bat`
  - SDK/Client: `pip`
  - Auth: None in repo configuration
- PyTorch CPU wheel index - fallback dependency installation path in `V2/install.sh` and `V2/install.ps1`
  - SDK/Client: `pip --index-url https://download.pytorch.org/whl/cpu`
  - Auth: None
- OS package managers - platform setup for SoX and Python support
  - SDK/Client: `brew` and `apt-get` in `V2/install.sh`, Scoop in `V2/install.ps1`, Chocolatey-aware flow in `V2/install.bat`
  - Auth: local OS privileges as needed

## File-Based Protocols

**Trigger and Response Files:**
- `V2/review_gate_v2_mcp.py` writes `review_gate_trigger.json` plus backup files `review_gate_trigger_0.json`, `review_gate_trigger_1.json`, and `review_gate_trigger_2.json` into `/tmp` on macOS/Linux or the system temp directory on Windows
- `V2/cursor-extension/extension.js` polls those temp files every 250ms, consumes matching trigger payloads, and opens the popup UI
- `V2/cursor-extension/extension.js` acknowledges popup activation by writing `review_gate_ack_<trigger_id>.json`; `V2/review_gate_v2_mcp.py` blocks on that acknowledgement before waiting for user input
- `V2/cursor-extension/extension.js` writes user replies to `review_gate_response_<trigger_id>.json`, `review_gate_response.json`, `mcp_response_<trigger_id>.json`, and `mcp_response.json`; `V2/review_gate_v2_mcp.py` reads and deletes the first matching file

**Speech Pipeline Files:**
- `V2/cursor-extension/extension.js` records WAV files as `review_gate_audio_<trigger_id>_<timestamp>.wav` in the temp directory
- `V2/cursor-extension/extension.js` requests transcription through `review_gate_speech_trigger_<trigger_id>.json`
- `V2/review_gate_v2_mcp.py` writes the result to `review_gate_speech_response_<trigger_id>.json` and cleans up completed audio files

**Status and Audit Files:**
- `V2/review_gate_v2_mcp.py` appends runtime logs to `review_gate_v2.log`
- `V2/cursor-extension/extension.js` checks the modification time of `review_gate_v2.log` to infer MCP health
- `V2/cursor-extension/extension.js` appends user activity to `review_gate_user_inputs.log`

## Platform-Specific Integration Points

**macOS/Linux:**
- Install target: `~/cursor-extensions/review-gate-v2/`, created by `V2/install.sh` and described in `V2/INSTALLATION.md`
- Cursor MCP config: `~/.cursor/mcp.json`, written by `V2/install.sh` and templated by `V2/mcp.json`
- Cursor rules directory: `$HOME/Library/Application Support/Cursor/User/rules` on macOS and `$HOME/.config/Cursor/User/rules` on Linux, managed by `V2/install.sh`
- Temp files: `/tmp/review_gate_*`, enforced by `get_temp_path()` in `V2/review_gate_v2_mcp.py` and `getTempPath()` in `V2/cursor-extension/extension.js`
- Shell integrations: `brew`, `apt-get`, `open -a Cursor`, and optional `cursor --install-extension` in `V2/install.sh`

**Windows:**
- Install target: `%USERPROFILE%\cursor-extensions\review-gate-v2\`, created by `V2/install.ps1` and `V2/install.bat`
- Cursor MCP config: `%USERPROFILE%\.cursor\mcp.json`, written by `V2/install.ps1` and `V2/install.bat`
- Cursor rules directory: `%APPDATA%\Cursor\User\rules`, managed by `V2/install.ps1` and `V2/install.bat`
- Temp files: `%TEMP%\review_gate_*`, reached through `tempfile.gettempdir()` in `V2/review_gate_v2_mcp.py` and `os.tmpdir()` in `V2/cursor-extension/extension.js`
- Windows launch paths: `cursor.cmd` and `Cursor.exe` lookups in `V2/install.ps1` and `V2/install.bat`; package-manager integration uses Scoop in `V2/install.ps1`

## Environment Configuration

**Required env vars:**
- `PYTHONPATH` - points Cursor’s MCP server process to the installation directory, as shown in `V2/mcp.json`
- `PYTHONUNBUFFERED=1` - keeps stdio/log output unbuffered for the MCP process configured by `V2/mcp.json`
- `REVIEW_GATE_MODE=cursor_integration` - mode flag written into MCP config by `V2/mcp.json`, `V2/install.sh`, `V2/install.ps1`, and `V2/install.bat`
- No secret-bearing env vars are referenced in the runtime code under `V2/review_gate_v2_mcp.py` or `V2/cursor-extension/extension.js`
